# 🚀 CUPED / Multi-CUPED / ML-CUPED

Данный проект представляет собой мощный аналитический инструмент для **снижения дисперсии (Variance Reduction)** в А/В-тестах.

В условиях реального бизнеса целевые метрики (выручка, конверсии, возвращаемость) часто обладают огромной дисперсией и «тяжелыми хвостами». Методы семейства CUPED используют предэкспериментальные данные (ковариаты) пользователей для того, чтобы «объяснить» и вычесть естественный шум, тем самым **кратно сокращая требуемый размер выборки и ускоряя Time-to-Market (TtM)**.

---

## 🌟 Ключевые возможности

Скрипт автоматически прогоняет целевые метрики через три архитектуры и выбирает лучшую:
1. **Single CUPED:** Классический одномерный метод, выбирающий историческую ковариату с максимальной линейной корреляцией к таргету.
2. **Vanilla Multi-CUPED (OLS):** Многомерный линейный подход. Использует линейную регрессию (`LinearRegression`) для отбора Топ-10 лучших предпериодных признаков по фолдам.
3. **ML-CUPED (Machine Learning CUPED):** Продвинутый нелинейный метод на базе градиентного бустинга (`CatBoostClassifier` / `CatBoostRegressor`).
   * 🛡️ *Включает строгую OOF (Out-of-Fold) кросс-валидацию (`KFold` / `StratifiedKFold` на 5 фолдов) для полного исключения утечки данных (Target Leakage) и переобучения*.
4. **Автоматический расчет MDE:** Скрипт на лету пересчитывает минимально детектируемый эффект (MDE) по Стьюденту.
5. **Интерпретация (XAI):** Поддержка `SHAP` для объяснения того, какие исторические признаки внесли наибольший вклад (Summary Plot, Dependence Plot).

## 🧩 Описание функций (Архитектура кода)

Код спроектирован по модульному принципу. Ниже приведено описание основных логических блоков:

1. **Хелперы и Валидация**
   `preprocess_data(df)`: Очистка данных, замена -1 на 0 и заполнение пропущенных числовых значений нулями.
   `is_binary_metric(df, target_col)`: Определяет, является ли метрика бинарной (конверсией), анализируя количество  уникальных значений.
   `estimate_mde(n_samples, var_metric, mean_metric)`: Рассчитывает относительный MDE (в процентах) по t-распределению Стьюдента (уровень значимости α=0.05, мощность 80%).
2. **Ядро алгоритмов (Variance Reduction)**
   `apply_vanilla_multi_cuped(...)`: Многомерный линейный подход. Обучает `LinearRegression` по K фолдам на Топ-10 предпериодных признаках, генерирует OOF-предсказания и рассчитывает оптимальный θ.
   `get_oof_predictions_catboost(...) и apply_ml_cuped(...)`: Самый мощный метод. Обучает `CatBoost` в цикле K-Fold кросс-валидации (со стратификацией `StratifiedKFold` для бинарных таргетов). Полученный вектор предиктов используется для вычисления θ и очистки таргета от шума.

3. **Анализ и визуализация (Сохранение в cuped_output_plots)**
   `visualize_stratification_tree(...)`: Строит и сохраняет визуализацию дерева решений (DecisionTreeRegressor) для умного разбиения клиентов на страты по предпериодным признакам[cite: 3].
   `run_shap_analysis_full(...)`: Обучает модель на всей выборке и строит SHAP Summary Plot и Dependence Plot для интерпретации ML-модели[cite: 3].
   `plot_cuped_variance_reduction(...)`: Генерирует два совмещенных графика (KDE распределение плотности и Boxplot размаха), наглядно демонстрируя сжатие дисперсии[cite: 3].
   `plot_sanity_checks(...)`: Строит Scatter plot и график калибровки по децилям для оценки монотонности предсказаний[cite: 3].

4. **Оркестратор**
   `run_full_segmented_analysis(...)`: Создает директорию для вывода, загружает (или генерирует синтетические) данные, сегментирует их (по признаку nfu_flg), перебирает таргеты, вызывает методы CUPED, рассчитывает проценты снижения дисперсии и выводит итоговый отчет в консоль.
